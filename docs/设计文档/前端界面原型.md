# 实验室多端Web库存管理系统 - 前端界面原型

## 1. 设计原则

1. **简洁性**：界面设计简洁明了，减少不必要的元素，突出核心功能
2. **一致性**：保持界面元素的一致性，包括颜色、字体、按钮样式等
3. **响应式设计**：采用响应式设计，适配不同终端设备
4. **易用性**：操作流程简单直观，降低用户学习成本
5. **视觉层次**：通过合理的布局和颜色区分，建立清晰的视觉层次
6. **可访问性**：确保界面对所有用户友好，包括残障用户

## 2. 整体布局结构

### 2.1 布局概览

系统采用经典的三栏式布局：
1. **标题区域**：页面顶部，显示系统名称
2. **侧边栏区域**：页面左侧，包含功能菜单
3. **数据显示区域**：页面右侧，显示主要内容

### 2.2 响应式设计

| 设备类型 | 布局调整 |
|----------|----------|
| 桌面端（≥1200px） | 三栏式布局，侧边栏固定，数据显示区域自适应 |
| 平板端（768px-1199px） | 三栏式布局，侧边栏可折叠，数据显示区域自适应 |
| 移动端（<768px） | 单栏式布局，侧边栏隐藏，可通过菜单按钮展开 |

## 3. 登录界面

### 3.1 功能描述

用户登录系统，验证身份后进入主界面。

### 3.2 布局设计

- **居中卡片设计**：登录表单居中显示，采用卡片式设计
- **简洁的登录表单**：包含用户名、密码输入框和登录按钮
- **响应式布局**：适配不同屏幕尺寸

### 3.3 视觉设计

- **背景色**：浅灰色（#f0f2f5）
- **卡片样式**：白色背景，阴影效果（box-shadow: 0 2px 8px rgba(0, 0, 0, 0.15)）
- **按钮样式**：蓝色主题（#1890ff）
- **输入框样式**：带有图标前缀，提升用户体验

### 3.4 交互设计

1. **输入验证**：实时验证用户名和密码是否为空
2. **登录反馈**：登录过程中显示加载状态，登录失败时显示错误提示
3. **键盘导航**：支持回车键登录

## 4. 主界面

### 4.1 功能描述

系统的主要操作界面，包含标题区域、侧边栏区域和数据显示区域。

### 4.2 布局设计

#### 4.2.1 标题区域

- **位置**：页面顶部，宽度100%
- **高度**：60px
- **内容**：系统名称"实验室库存管理系统"
- **背景色**：深蓝色（#001529）
- **文字样式**：白色，居中显示，字体大小24px，行高60px

#### 4.2.2 侧边栏区域

- **位置**：页面左侧，宽度250px
- **高度**：100%（减去标题区域高度）
- **背景色**：白色（#ffffff）
- **边框**：右侧1px灰色边框（#f0f0f0）
- **内容**：
  - 系统Logo
  - 功能菜单（可折叠）
  - 用户信息和退出登录按钮

#### 4.2.3 数据显示区域

- **位置**：页面右侧，宽度自适应
- **高度**：100%（减去标题区域高度）
- **背景色**：浅灰色（#f5f5f5）
- **内容**：根据侧边栏选择的功能，显示相应的内容

### 4.3 视觉设计

- **侧边栏菜单**：
  - 菜单项高度40px
  - 选中状态背景色（#e6f7ff）
  - 鼠标悬停背景色（#f0f0f0）
  - 菜单项文字颜色（#333333）
  - 选中状态文字颜色（#1890ff）

- **数据显示区域**：
  - 内容区域白色背景
  - 卡片式设计，带有阴影效果
  - 清晰的标题和分隔线

### 4.4 交互设计

1. **侧边栏折叠**：点击折叠按钮，侧边栏宽度变为64px，仅显示图标
2. **菜单切换**：点击菜单项，数据显示区域切换到相应的内容
3. **面包屑导航**：显示当前页面的路径，方便用户返回上一级
4. **响应式调整**：在不同屏幕尺寸下，自动调整布局

## 5. 表格定义界面

### 5.1 功能描述

管理员定义和管理库存表格的结构，包括总表名和列信息。

### 5.2 布局设计

- **位置**：数据显示区域
- **布局**：垂直布局，包含总表名输入、列定义和操作按钮
- **响应式设计**：在小屏幕设备上，输入框和按钮宽度自适应

  - 删除列按钮：红色（#ff4d4f）
  - 保存按钮：蓝色（#1890ff）

### 5.4 交互设计

1. **总表名输入**：实时验证总表名是否为空

   - 点击"添加列"按钮，在列表末尾添加新列

4. **保存反馈**：保存过程中显示加载状态，保存成功后显示提示信息

## 6. 已定义表格管理界面

### 6.1 功能描述

显示所有已定义的表格列表，支持查看、编辑和删除操作。

### 6.2 布局设计

- **位置**：数据显示区域
- **布局**：

## 7. 数据管理界面

### 7.1 功能描述

管理选定表格的库存数据，支持添加、编辑、删除和查询操作。

### 7.2 布局设计

- **位置**：数据显示区域
- **布局**：
  - 顶部：表格信息和操作工具栏
  - 中部：数据过滤和搜索区
  - 下部：数据表格和批量操作栏
  - 底部：分页控件

### 7.3 视觉设计

- **操作工具栏**：白色背景，包含添加数据、导入数据、导出数据和刷新数据按钮
- **数据过滤区**：浅灰色背景（#f5f5f5），包含搜索框和高级过滤按钮
- **数据表格**：
  - 表头背景色：浅灰色（#f0f0f0）
  - 表格行交替背景色：白色和极浅灰色（#fafafa）
  - 操作按钮：查看、编辑、删除（根据用户角色显示）
- **批量操作栏**：蓝色背景，包含批量删除按钮

### 7.4 交互设计

1. **数据添加**：
   - 点击"添加数据"按钮，弹出数据添加表单
   - 表单字段根据表格结构动态生成
   - 保存后自动刷新表格

2. **数据编辑**：
   - 点击"编辑"按钮，弹出数据编辑表单，预填当前数据
   - 保存后自动刷新表格

3. **数据删除**：
   - 点击"删除"按钮，显示确认对话框，确认后删除数据
   - 支持批量选择和批量删除

4. **数据查询**：
   - 全局搜索：搜索表格中的所有字段
   - 高级过滤：根据列名设置过滤条件
   - 排序：点击列标题进行排序

5. **导入导出**：
   - 支持CSV格式数据导入
   - 支持CSV格式数据导出

6. **分页功能**：支持页码跳转和每页显示条数调整

## 8. 报表统计界面

### 8.1 功能描述

生成和查看库存数据的统计报表，支持按不同维度进行统计和分析。

### 8.2 布局设计

- **位置**：数据显示区域
- **布局**：
  - 顶部：报表查询条件
  - 中部：统计图表和数据表格
  - 下部：导出按钮

### 8.3 视觉设计

- **查询条件区**：白色背景，包含表格选择、时间范围和统计维度选择

  - 表格行交替背景色：白色和极浅灰色（#fafafa）
- **导出按钮**：蓝色主题（#1890ff）

### 8.4 交互设计

1. **报表查询**：
   - 选择要统计的表格
   - 设置时间范围
   - 选择统计维度（如时间、人员、类型等）
   - 点击"生成报表"按钮，显示统计结果

2. **图表交互**：
   - 支持图表类型切换
   - 支持悬停查看详细数据
   - 支持点击图表元素过滤数据

3. **数据导出**：
   - 支持导出统计图表为图片
   - 支持导出统计数据为CSV或Excel格式

## 10. 响应式设计规范

### 10.1 断点设计

| 断点 | 屏幕宽度 | 布局调整 |
|------|----------|----------|
| 大屏幕 | ≥1200px | 三栏布局，侧边栏固定 |
| 中屏幕 | 992px-1199px | 三栏布局，侧边栏固定，数据显示区域宽度调整 |
| 小屏幕 | 768px-991px | 侧边栏可折叠，数据显示区域宽度自适应 |
| 超小屏幕 | <768px | 单栏布局，侧边栏隐藏，菜单按钮触发 |

### 10.2 布局调整规则

1. **标题区域**：始终固定在顶部，高度不变

2. **侧边栏区域**：
   - 大屏幕：固定显示
   - 中屏幕：固定显示，宽度调整为200px
   - 小屏幕：默认折叠，可通过按钮展开
   - 超小屏幕：默认隐藏，可通过顶部菜单按钮展开

3. **数据显示区域**：
   - 大屏幕：宽度自适应
   - 中屏幕：宽度自适应
   - 小屏幕：宽度100%
   - 超小屏幕：宽度100%

4. **表单元素**：
   - 大屏幕：水平排列
   - 小屏幕：垂直排列

5. **表格**：
   - 大屏幕：完整显示所有列
   - 小屏幕：显示核心列，其他列可通过展开查看

## 11. 交互设计规范

### 11.1 反馈机制

1. **加载状态**：
   - 点击按钮后，显示加载动画
   - 数据加载时，显示骨架屏或加载提示
   - 页面加载时，显示加载进度条

2. **操作结果反馈**：
   - 操作成功时，显示成功提示（绿色）
   - 操作失败时，显示错误提示（红色）
   - 操作警告时，显示警告提示（黄色）
   - 提示信息自动消失或可手动关闭

3. **悬停效果**：
   - 按钮、链接等可点击元素，悬停时显示状态变化
   - 表格行悬停时，背景色变化
   - 菜单悬停时，显示子菜单或提示信息

### 11.2 导航设计

1. **面包屑导航**：
   - 显示当前页面的层级关系
   - 支持点击返回上一级页面

2. **侧边栏导航**：
   - 高亮显示当前选中的菜单项
   - 支持多级菜单
   - 支持折叠和展开

3. **顶部导航**：
   - 显示系统名称
   - 显示用户信息和退出按钮
   - 移动端显示菜单按钮

### 11.3 表单设计

1. **输入验证**：
   - 实时验证输入内容
   - 显示清晰的错误提示
   - 支持键盘导航

2. **表单布局**：
   - 标签居左或居上
   - 输入框宽度适中
   - 提交按钮居中或右对齐

3. **动态表单**：
   - 根据业务规则动态显示或隐藏字段
   - 支持动态添加和删除字段
   - 支持字段排序

## 12. 可访问性设计

1. **键盘导航**：
   - 支持键盘Tab键导航
   - 支持键盘快捷键
   - 焦点状态清晰可见

2. **屏幕阅读器支持**：
   - 添加适当的ARIA属性
   - 确保所有内容可被屏幕阅读器识别
   - 提供替代文本

3. **颜色对比度**：
   - 确保文字与背景的对比度符合WCAG标准
   - 提供高对比度模式选项

4. **字体大小**：
   - 支持浏览器字体大小调整
   - 避免使用固定字体大小

5. **操作反馈**：
   - 提供视觉和听觉反馈
   - 确保所有操作都有明确的结果提示

## 13. 总结

本前端界面原型设计遵循了简洁、一致、响应式和易用性的设计原则，详细描述了系统的主要界面、布局、视觉设计和交互设计。设计考虑了不同角色用户的需求和操作习惯，同时兼顾了系统的可扩展性和可访问性。

前端界面原型为开发人员提供了清晰的设计指导，确保开发出的系统符合预期的视觉效果和用户体验。同时，设计也考虑了未来功能扩展的可能性，便于后续添加新功能和优化现有功能。
